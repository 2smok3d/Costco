Item Lookup
IAIItem Alpha Inquiry✓▼

The single most-used screen. Gateway to ~80% of all merchandise data. Search any item by keyword, item #, dept, UPC (drop check digit), location group, or category code.

Fields
  • Description — keyword search (most common). Partial words work.
  • Dept — narrow by dept number (e.g., 61 for Meat, 65 for Produce)
  • Location — defaults to home warehouse; never leave blank
  • Location Group — *NW (Northwest), *SW (Southwest), *USALL (all US), *CN (Canada), *STAZ (Arizona state), etc.
  • Category Code — use with Dept to filter (e.g., TV category in Dept 24)
  • Display On Hand: Y = in-stock items only; N = all ever carried

ITMWItem Inquiry for Warehouse ⭐✓▼

Most detailed item screen, scoped to YOUR warehouse. Real-time data updated throughout the day. Shows on-hand, sell price, units on order, next delivery ETA, NSI on hand, today's sales.

  • Enter item # directly or drill in from IAI opt 8
  • NSI = Non-Saleable Inventory (damaged/held stock)
  • Today's Sales timestamp shows how recent the data is
F9 RebatesF14 TaxesF21 Print SignF22 EBT / Env Fees
Add 2 days to next-delivery ETA when telling members — supply delays are real. Item info is entered by the buying team, warehouse data updates periodically throughout the day.
IILItem Inventory Lookup (Multi-Warehouse)✓▼

On-hand across multiple warehouses. Redirect members, source stock-outs via transfer-in, or benchmark your warehouse vs. peers on focus items (stamps, movie tickets, hot SKUs).

  • Enter item # → use scope: region code or wildcard *MW, *USALL, *NW, *SW, *STAZ
  • Returns: each warehouse, on-hand qty, on-order, status
  • Also shows sales ranking across region (great for competitive context)
⚠ IIL updates ONCE OVERNIGHT in QBATCH. Never promise real-time accuracy to a member — data is from yesterday.
UPCUPC / Barcode Lookup✓▼

Find an item by its 12-digit GS1 barcode when scanning fails at the register. Enter the barcode number to get item description and item number.

Always DROP the last digit (check digit) when keying a UPC manually — Costco convention. Same rule applies when keying IRC coupon codes.
Variable-weight items (Meat, Deli, Bakery): UPCs start with "2". First 6 digits = item ID, trailing digits encode price or weight per GS1 retail convention.
RHIReceiving History Inquiry✓▼

Complete receiving history for a specific item: every PO and delivery with date, vendor, PO #, quantity, and cost. Essential for investigating receiving errors and validating vendor claims.

  • Enter item # + warehouse
  • Page through all deliveries in reverse-chronological order
  • Cross-reference against physical count to catch vendor short-ships
Catching receiving discrepancies same day is 5× easier than next day. Run after every meat truck lands. The most common reason "the cooler doesn't tie out" is a vendor short-ship posted at full count.
DASHReceiving Dashboard✓▼

Inbound truck dashboard — every scheduled receiving appointment for the day: PO #, vendor, ETA, status, door assignment. Receiving clerks live here during morning shifts. Other depts can view their incoming inventory here too.

Cross-reference DASH against RHI for any stocked-out SKU — the next truck may already contain it. For full processing instructions, refer to the Receiving Manual.
IXIItem / UPC Cross Reference✓▼

Cross-reference an item number with all associated UPCs. Primary use: when an item won't scan at the register — if the UPC is set up in IXI but not scanning, run an EPC item download to push the fix to registers immediately.

RF Option 88 = same function on scanner
IRCIInstant Rebate Coupon Inquiry✓▼

View active, expired, or future IRCs. Primary uses: find coupon # for register adjustments, check active savings on items, diagnose why a coupon failed at the register.

  • Filter by location group, location, and/or dept/item #
  • Place X next to Active, Expired, or Future
  • Expired: must enter start date; end date defaults to today
  • Future: enter both start AND end date (start must be future date)
  • Screen shows: item #, coupon #, IRC amount, qty limit, start/end dates, units on hand, units in transit
When keying an IRC coupon # at the register: DROP the last digit OR precede number with </>. Manufacturer-funded vs. Costco-funded flag also visible here.
SISign Inquiry (View Without Printing)✓▼

View a price sign's full content without printing: sign size, price, instant-savings text, IRC details, and effective dates. Verify before printing to avoid wrong-sign waste.

IGMItem Group Maintenance✓▼

Create and maintain item groups for use in ISI and COMP. Only active warehouse items can be added. Groups persist for future use.

  • Enter warehouse # and group name → ENTER
  • Add/remove item numbers from the group → ENTER to confirm
  • F23 to delete the entire group
RF Option 19 = same on scanner
ISIItem Sales Inquiry✓▼

View sales for individual items or an IGM group. Enter item numbers one by one, or use a named IGM group to pull all items at once.

The IND256 report generated by ISI prints to the MAIN office printer regardless of which terminal requests it.
ITMGItem Group Maintenance (User Groups for COMP)✓▼

Create user-defined item groups for COMP comparison analysis. F6 to add a new group; can create, edit, or link to other user groups. Useful for tracking multi-item programs (e.g., charity donation items).

MACRMerchandise Audit Count Report✓▼

Print a full department inventory listing: every item with on-hand qty, location, category, and NSI on-hand. Used for cycle counts, period-end reconciliation, and buyer walk-throughs.

  • Enter dept # to print all items
  • Add Category Code to filter (e.g., TVs only in Dept 24)
RF Options 20/21/22 = count entry, clear, and print on scanner
Warehouse Sales & Analytics
DSIDaily Sales / Inventory Inquiry✓▼

Dept sales and inventory by dept, warehouse, region, or company. Daily, weekly, period, and YTD totals. NOT real-time — shows next day only.

  • By Warehouse: all depts for one location — most common use
  • By Department: same dept across all warehouses (US and CN) — compare against neighboring locations
  • Enter date in Date field; same-day data will not populate
DFIDaily Flash Sales Inquiry✓▼

Flash sales summary — daily, weekly, period-to-date, and YTD for each warehouse. Not real-time (shows next day). Use F prompts at bottom for additional detail views.

DSLDepartment Sales Listing ⭐✓▼

Daily, weekly, and YTD sales by department with vs.-last-year and vs.-plan columns. The number you bring to every period-end meeting. Core tool for trend analysis and buyer conversations.

Know DSL for your dept every shift. Pair with IIL: if you're running +20% YoY but haven't adjusted your order, you'll stock out. Frame all recommendations with DSL numbers: "DSL shows -22% YoY; recommend cutting 2 pallets from the next PO."
STKSales Tracking Inquiry — 4-Week ⭐✓▼

4-week unit sales and Damaged & Destroyed (D&D) by department item. Critical for fresh production planning — shows sales velocity and loss patterns over 4 weeks.

  • Enter fiscal year, period, week, dept, and warehouse
  • F4 for Category Inquiry to filter
  • F19/F20 to scroll left/right across the 4 weeks
  • F21 to print the full report
  • F22 to export to Excel for further analysis
For Meat (Dept 61): high Days of Supply (DOS = on-hand ÷ avg daily units) on any item means over-production → mark down before pull date. A sudden spike in D&D on a fresh item = cutters are over-producing or there's a quality issue.
IATOPTop Items by Department✓▼

View and compare top items for sales, refunds, adjustments, D&D, and markups/markdowns. Compare a single location or two locations/regions for a specified time frame.

  • Enter warehouse, Dept (blank = all depts), date range, fiscal period/year
  • Default top 10 — change as needed
  • Add Category Code with a dept to narrow (e.g., top TV sales in Dept 24)
  • F19/F20 — toggle between comparing top ranks and top items across warehouses/regions
Run IATOP before every buyer or regional walk-through. Pair with IIL: if you're #1 in sales but low in regional on-hand, stock-out is imminent.
IAWDWarehouse Dashboard✓▼

All-in-one warehouse metrics: inventory, membership, sales, and front-end productivity. Daily, weekly, period, and YTD. Not real-time — same-day data will not populate.

ViewContents
00 — MainMajor warehouse statistics for selected date/week/period/year. Change date or warehouse as needed.
01 — InventoryOn-hand and sales in $ for each dept (this year vs last year). Total on-hands, receivings, D&D, other inventory stats.
02 — SalesMajor sales stats per dept: sales $, penetration % to warehouse, comp % vs LY, for selected date/week/period.
03 — Front EndSimplified IAFEP view — major FE productivity stats for the warehouse.
IAIMIInventory Movement Summary Display✓▼

Stock status summary by company, region, warehouse, or dept level. Must include dept # alongside location for dept-level view.

  • Select an item → press F15 (Sales/UPC) to see UPC activity
  • UPC Activity view shows: how many times item was hand-keyed vs. each UPC scanned
UPC Activity is useful when investigating excessive hand-keying on an item — indicates a scanning issue that may need an IXI check and EPC download fix.
IADItem Activity by Department (D&D in Dollars)✓▼

On-hand adjustments OR destroyed items in dollars for a selected dept, fiscal year, period, and/or week. Shows each item's net landing cost, sell cost, and total extended cost and sell for the adjustment/D&D bucket.

  • Enter DS for Destroyed, or on-hand adjustment type
  • Enter warehouse, dept, time frame
ISOItem Sales by Operator ID✓▼

View sales for a specific item broken down by clerk/operator. Shows operator #, total units sold, and total dollars sold for the selected warehouse, date range, and item.

COMPComparison Analysis✓▼

Compare sales history by item, item group, dept, vendor/suffix, location, location group, category, or date range across the last 3 fiscal years. Uses ITMG groups for multi-item comparisons.

CDCSICost Dept Category Sales Inquiry (IMU)✓▼

View and work with cost department margin components by location, dept, and category. Primary use: view the Initial Markup Percent (IMU) for specific items within each cost dept.

IWSWeekly Item Unit Sales (Hourly Breakdown)▼

Sales for every hour of the day for specified item numbers. Enter warehouse and time frame. Page Down for multiple item numbers.

WIUSHalf-Hour Item Unit Sales▼

30-minute interval version of IWS. Enter warehouse, item number, time frame.

SHCSales History by Category▼

Sales history for any category. Enter Dept, Warehouse, Time Frame, Category (F5 for Category Inquiry). F4 to break into weekly sales.

SLHSales History by Item▼

Sales history for a specific item. Enter warehouse, time frame, item number. F4 to break into weekly view.

SPISupply Purchase Inquiry▼

Shows what items were purchased for a specific department. Enter warehouse, time frame, then key 1 next to dept accounts to view. Key 1 next to each day for item-by-item daily purchases.

Signs & Merchandising
EDSREvery Day Sign Request (Print Sign)✓▼

Print a price sign when you know the item # and don't need manual changes. Most buildings have two sign printers (1 = main, 2 = mobile).

  • Enter option EDSR → select sign size (S=small, M=medium, L=large)
  • Enter up to 45 item numbers or UPCs
  • Press F21 (Shift+F9) to print
  • Select printer 1 (designated) or 2 (mobile)
Walk the floor every morning with an EDSR printout — comparing paper to product is how you catch price mismatches before members do. Most common sign size for daily use: M (medium) or the size designated by your warehouse.
RF Option 23 = print signs from the scanner
EDMPPrint Daily Maintenance (All Price Change Signs)✓▼

Batch-prints all signs where buyer maintenance changed the price/content. Updated at 3:00am local time — do not run before that.

  • Enter option EDMP after 3:00am
  • Filter: on-hand items OR price changes; current date auto-populates (change for holiday planning)
  • Set Print = Y, choose size (default M/medium), press F21
  • Signs auto-send to sign printer 1
Signs MUST be printed and hung before opening every day. For holiday closures, change the date to that holiday to ensure no price-changed signs are missed.
ESPCSign Print Configuration✓▼

Configure default settings for EDMP: department default sign sizes, which depts print with daily maintenance. Press F10 (Change) to adjust settings.

EDSMSign Maintenance — Custom / Free Form Signs✓▼

Customize a sign or create a free-form sign. Use for "last one display," price corrections not yet downloaded, or description changes that won't load until tomorrow.

  • Enter item #, size, quantity → edit description, price, or other fields
  • Leave item # blank for a completely blank free-form sign
  • Signs print AFTER you exit (F3) — not immediately on the edit screen
If you change the sell price, the per-oz/per-bottle unit price won't auto-update — calculate manually. Demo profiles do not have access to EDSM.
EPCEmergency Price Change + Item Downloads + Blocks✓▼

Mark down items, correct register prices, force item downloads, and block items from sale. Access typically limited to staff managers — assigned via WPM.

  • Enter option EPC → enter User ID and WPM password
  • F6 to add new EPC → enter item #, new price, and reason → ENTER to apply
  • To copy an EPC from a prior day: change the date, key 3 next to the EPC
Item Downloads (F15)
  • Press F15 in EPC to access item downloads
  • Enter fields → F8 to update — forces immediate sync to registers
  • Use when: price mismatch between AS400 and register, buyer adds a fee mid-day, or to restore original price after an EPC (EPC cannot mark price back up — only item download can)
Item Blocks (F18)
  • Press F18 in EPC to view/manage blocked items
  • K = key flick required at register; P = item prohibited from sale entirely
  • F6 to add new block → enter K or P + reason
  • To delete: same process but key D next to the item
  • Block types appear on register as "NOT FOR SALE"
Regional rules may restrict who receives EPC access. "Not For Sale" on a register = check EPC F18 for an active block.
EPCIPrice Change Inquiry (Read-Only)✓▼

Read-only view of all active EPCs, item downloads, and blocked items. No editing — pure inquiry.

SNMSerial Number Maintenance (High-End Watches)✓▼

Replaced the paper Watch Tracking Log. Record all high-end watch transactions: receivings, sales, refunds, and RTD shipments.

  • Enter option SNM → select transaction code
  • Enter item #, serial #, or membership #
  • Every watch movement must be recorded here
SNISerial Number Inquiry (Watch Audit)✓▼

Audit tool for SNM. Discrepancy View (most common) shows any mismatches between SNM movement records and current on-hand inventory.

DGMDepartment Group Maintenance (Hot Sheet Order)✓▼

Configure which departments appear on the hot sheet and their print order. Sequence numbers 1–4 determine print priority (1 prints first). Enter 0 to remove a dept from the hot sheet.

KRIRoadshow Inquiry✓▼

Roadshow vendor schedule: who's in the building, when, and where on the floor.

Transaction Lookup
HISTORYShopping History — Privacy Rules (US)✓▼

Member transaction history in WII, WIIA, and SAST is limited to 10 years from original transaction date. Strict privacy rules apply.

RequestAllowed?
Member requesting their own historyYes — valid state ID must be verified in person
Spouse/household member requesting another'sNo — must request their own
Business owner requesting affiliate's historyNo — affiliates must request themselves
NEVER turn the screen to show a member any information. History 10+ years old shows with member # 99 when searched by warehouse + item #. Refer to Shopping History – US memorandum for full policy.
WIIWarehouse Item Sales Inquiry (Member Purchase History) ⭐✓▼

The refund desk's primary tool. Look up transactions for a specific member or warehouse — includes purchases and refunds. 10-year history.

  • Enter Member # PLUS either Dept OR Item # (one required — not both)
  • Clear the Location field for broadest results across all warehouses
  • Modify start/end dates to search a specific range
  • Refunds show as negative amounts in the Ext Sell Price column
Key Functions
  • F11 = Fold — adds item descriptions (helpful when only description is known, not item #)
  • ENTER on a transaction — drills into full transaction detail
  • F21 = Print a transaction receipt
  • White-highlighted items — item had an IRC when purchased; press ENTER to see coupon details
WII does NOT include Costco.com online orders — those live in EOI/CNI. Same-day transactions are not available here — use the EDP controller for same-day lookup.
EOI / CNIEcommerce Order Inquiry / BOPIW Check-In✓▼

Research Costco.com (EOI) and Costco.ca (CNI) orders. Also used to check in Buy-Online-Pickup-In-Warehouse orders.

  • Filter by Order # (9–10 digits) or Bill-to Membership #
  • ENTER → Order Detail screen (items, order #, member #, date, total)
  • Key 5 or X next to an item → item detail (price, S&H, tax, coupon amount)
BOPIW Check-In Functions
  • F4 — prompts Serial Number field
  • F8 — confirms the update
  • F9 — marks member pickup
Coupons on Ecom are treated as tender — they appear in item detail, not as a separate line like in-store IRC adjustments. Customer Notes and Status Description via F prompts at bottom.
BDOIBusiness Delivery Order Inquiry✓▼

Detailed info for business delivery orders. Similar to EOI. Search by order number or membership number. Shows items purchased, order totals, and any adjustments.

  • Key 5 or X next to item → individual item/price adjustments
  • Note: first screen shows total adjustments only — must drill in for per-item detail
F14 Payment methodF19 Refunds on order
KOMKiosk Order Maintenance (Dept 44 Special Events)✓▼

View Dept 44 (Special Events) purchase info. Main use: process refunds when the vendor picks up the merchandise and the member has nothing to return.

  • Filter by order #, membership #, phone #, item #, or warehouse #
  • For Dept 44 refunds with no item to return: look up in KOM first, then process refund
Refer to Department 44 – Special Events Refunds job aid for full procedures.
RLGMember Refund Log✓▼

View refund transaction details. Defaults to home warehouse and previous day — both fields can be changed.

  • Apply filters → ENTER to display matching refunds
  • Y in Dept Ring column = refund performed as department ring
  • * in Dept Ring column = department ring converted to inline item
  • Key X next to operator number → full refund detail screen
Tracking number column only displays when NOT viewing same-day data.
SASTSales Audit (Transaction Search)✓▼

Find transactions on a specific register for a date range. Multiple filters can be combined simultaneously: register #, date range, transaction type, tender type. Also accessible from within WII.

  • Set filters → ENTER to list matching transactions
  • TAB to a transaction → press Field ENTER to drill in
  • Items grouped by dept/item # in ascending order (NOT scan sequence)
SAST does not preserve the order items were scanned at the register. For exact scan sequence, use the EDP controller.
TAXISales Tax Inquiry (Prior Year Member Total)✓▼

Shows total sales tax a member paid during the previous calendar year. Used when members need documentation for state sales tax federal deductions.

  • Enter member number → view prior calendar year tax total
Refer to Sales Tax Inquiry – US memorandum for full details and applicable states.
Profiles & Passwords

  • Enter option EPRFM → F4 for position type list
  • Enter employee # → ENTER → F6 to assign user
  • Tab to User Template → F4 to select template → F8 to finalize
  • To disable: enter position type + employee # → ENTER → confirm info → F23 twice
Combo Employees (Multi-Dept)

  • Employees may only have ONE AS400 profile
Full-time Sales Auditors CANNOT carry any other profile — Sales Audit is exclusive. Verify the employee's JTS job code is entered — this auto-triggers Costco U training requirements.
EPRFDEmployee Profile Display (Read-Only)✓▼

Inquiry-only. View profiles assigned through EPRFM. Use to audit access, troubleshoot "I can't see X menu," and prep for compliance reviews.

  • Enter WPM → F6 to add → enter User ID (max 10 chars) and name
  • Employee sets their own password

Labor Scheduling
LSDTDaily Sales Tracking (Sales + Labor) ⭐✓▼

Primary FE tool for checking hours scheduled vs. hours needed, with near-real-time sales. Available to anyone with an AS400 sign-on; shows home warehouse only.

FieldDefinition
Actl LaneTotal Front End registers open during the time interval
Sched LaneLanes scheduled based on # of employees designated as cashier
Hrly Actual SaleActual sales from Front End registers
Hrly Plan SalePlanned sales from LSWM — rolling 4-week average × daily sales plan
Sls DiffCuml Act Sale – Cuml Plan Sale — running variance for the day
Use to decide whether to call people in or send people home. View intervals in 15, 30, or 60-minute periods.
LSCMLabor Schedule Control File Maintenance✓▼

Maintains the control values used by the entire Labor Scheduling system. Available to Staff Managers, above, and Front End profiles. Sets: $ per labor hour, number of registers, opening and closing hours. Calculates recommended scheduled hours based on sales plans.

Must be updated for holiday hours (Christmas Eve, New Year's Eve, etc.) to reflect early or extended opening and closing times — affects forecasting accuracy.
LSWTWeekly Sales Tracking✓▼

Same metric set as LSDT, rolled up to the week with daily breakouts. Basis for building next week's schedule.

LSWMEnter Projected Sales (Labor Plan)✓▼

Enter projected (planned) sales — updates the planned sales figure in LSDT and directly affects the number of suggested scheduled hours. Available to GMs, AGMs, FRNs, and ADMs.

  • Plans can be changed up to and including the day of sales
  • Red # in "Imp Flg" field = sales impact flagged for that day (large resale order, kiosk, jewelry)
  • To adjust hours for early closures (Christmas Eve, New Year's Eve): key 6 next to the date → change "Select a Plan" field to 3 → set Selected Date to same holiday previous year
LSDCDoor Count Entry✓▼

Enter door counts throughout the day. Gives supervisors an accurate representation of how many lanes should be open. Combined with LSDT gives average ticket per member — a key KPI.

LSIMSales Impact Entry (Manual Impacts)✓▼

Enter unusual sales impacts for the day that will affect next year's sales planning. Examples: large resale cigarette orders, large jewelry purchases, large returns.

  • F6 to add impact → enter reason and dollar amount
  • Change the date at top to view impacts for other dates
  • Auto-populated impacts (no manual entry needed): resale, refunds, kiosk orders, gasoline, jewelry, special events, coupons
LSRMLabor Schedule Rule Maintenance (Dollar Caps)✓▼

Create dollar-limit rules per department. Member transactions exceeding the limit are excluded from sales figures when the system forecasts future sales/lanes. Prevents large outlier transactions from skewing staffing recommendations.

These transactions still appear in LSDT and LSWT as actual register sales — the rule only affects forecasting.
LSSILabor Schedule Lane & Shift Coverage✓▼

View system-forecasted lanes needed vs. lanes scheduled for a specific day and time. Use to identify where adjustments are needed.

System assumes ALL Service Clerks scheduled will function as cashiers.
Membership
MBA9Membership Inquiry (Read-Only)✓▼

View-only membership lookup. Quicker and safer for routine verification — cannot edit membership data. Also shows forecasted membership fees.

  • Toggle search mode with F11: Membership # → Last/First Name → "Sounds Like" → ID type + # → 7-digit phone → mailing address
  • Once found: F4 to see available action code prompts

Key Identifiers You Must Know

Item numbers: 6–7 digit numeric codes (older items: 5–6 digits). Same number used by employees, members, vendors, and signage.

UPCs: 12-digit GTIN-12 at consumer pack level, 14-digit at case/pallet. Drop last digit (check digit) when keying manually — always.

Variable-weight UPCs (Meat/Deli/Bakery): Start with "2". First 6 digits = item ID, trailing digits encode price or weight. Set by Bizerba/Hobart scale-printer.

Warehouse numbers: 3-digit numeric + 2-letter region code. Tempe = SW region. Newer warehouses are 1000+.

Costco terminology: Warehouse (not "store") · Depot (not "DC") · DRS (Daily Register Sheet) · RTV (Return To Vendor) · IRC (Instant Rebate Coupon) · SIT (Supervisor In Training) · MPU (Merchandise Pick Up) · Combo Employee (multi-dept) · D&D (Damaged & Destroyed)

How Data Flows

Costco's IBM i (AS400) runs three subsystems:

QINTER — Your interactive 5250 terminal session.

QBATCH — Overnight POS rollups, replenishment runs, sign-print batches, cross-warehouse inventory refresh (IIL).

QSPL — Printed signs, receiving manifests, Daily Register Sheets.

POS transactions update DB2/400 in near-real time. But cross-warehouse on-hand (IIL) and sales-rank screens (IATOP) refresh nightly in QBATCH — never promise same-day accuracy.

Daily Routine — Fresh Dept Lead (Before Doors Open)

1. Sign on → DSL — yesterday's dept sales vs. plan vs. LY

2. LSDT — sales + labor variance (call in? send home?)

3. RHI — confirm any item received yesterday posted correctly

4. Spot-check IAI / IIL on your top-10 SKUs for OOS risk

5. Walk the floor with EDSR printout — verify every sign matches its item

6. Build morning-huddle talking points: one win, one problem, one action

Closing Routine

1. Final DSL / LSDT read — how did the day close?

2. WRKSPLF — clear and route the day's spool files

3. Key shrink / damages / donations into the shrink entry module (ask trainer for screen name)

4. Key production counts if applicable (production sheet → AS400 entry)

5. Sign off cleanly with SIGNOFF

Weekly Routine

Mon: 7-day DSL by SKU — rank top-25 and bottom-25. Identify slow movers with cover days >5.

Tue: Weekly RHI variance review for your truck cycle.

Wed: Comp-shop maintenance — tobacco, liquor, seasonal price audits.

Thu: EDSR review for upcoming weekend pricing. Check EDMP for Saturday/Sunday price changes.

Fri: Pre-weekend OOS sweep. Place transfer-in requests with sister warehouses via IIL.

Sun/EoW: Print weekly DSL spool, file in dept binder, prep weekly variance commentary for manager.

Period (4-Week) Routine

3 days before close: Reconcile all RHI receivings. Post all damages, donations, transfers. Reconcile in-transit pallets. Run FFPII to verify on-hands.

Count day: Print expected on-hand from IAI opt 1 for every dept SKU → physical count → key variances. Biggest variances in slow-mover bins.

Day after close: Review shrink % vs. company target (0.11–0.12%). Document root cause for every variance over threshold. Print period DSL + turn calc + shrink summary — one-pager for AGM/GM.

Day 1 new period: Re-baseline cover days. Reset standing orders if YoY trend shifted ±10% or more.

Costco runs 13 fiscal periods per year — one per 4-week cycle. 13 physical inventories per year for fresh depts.

Buyer Language — The Justify-an-Order Template

"Item _____, Dept ____, showed cover days of N (target ~3 days for fresh). DSL shows -X% YoY. RHI shows we're still receiving at last year's rate. Recommend cutting the standing receive by Y cases. Estimated annual savings in shrink: $_____."

Buyers and AGMs reward this structure because it speaks Costco's internal language: DSL (trend), RHI (receiving pattern), cover days (risk), dollar impact (justification).

Speak in dollars, not units: 30 extra cases of ribeye = $X tied up + spoilage risk. Print the spool file and bring paper to the meeting.

Error Messages & Fixes
CPF5032
Record already locked to this job
F12 to back out. If persistent: sign off and back on. IT can release via WRKOBJLCK if it's blocking.
CPF4101
File not found / level check on open
Run DSPLIBL — check if a library is missing from your list. Usually IT pushes a fix; sign back on after confirmation.
CPF5027
Record being used by another job
Wait and retry. If it's blocking critical work, IT can ENDJOB the offending session.
CPF2189
Job not authorized to use library
Your profile lacks access to a required library. Manager contacts IT to grant access.
CPF9999
Function check — unhandled program error
Press F3 or Enter. Run DSPJOBLOG and capture the exact CPF code. Send to IT — they need it to diagnose.
X II / II
Input Inhibited — keyboard won't respond
Press Reset key (left-Ctrl). Still stuck: System Request → 2 → Enter to end the request. Last resort: close and relaunch the emulator.
Terminal Secured
Screen locks after 90 seconds idle
Enter your operator # and password normally. Not an error — this is expected security behavior.
Job already exists for this user
Previous session still active
System Request → option 4 (from a second session) to read messages and end the old job cleanly.
U008
Register communication error
Power register OFF — hold power button 10 seconds. Wait 10 more seconds. Power ON — register reloads automatically.
Z001 / Z002
Register address error
Enter register # as 4 digits (register 1 = 1001, register 16 = 1016, register 501 = 1501). Press S2 (hold CTRL + RESALE + release).
NOT FOR SALE
Item blocked at register
Go to EPC → F18 to view blocked items. Check if a K (key flick) or P (prohibit) block was applied. D to delete if incorrect.
Spool stuck WTR/RDY
Print job won't print
WRKSPLF → option 6 to release, or option 2 to change OUTQ to a working printer. Check RDPRT/SPRT if rerouting is needed.
Password Disabled
3 bad sign-ons or 90+ days inactive
Manager contacts IT for a QSECOFR-level reset. Set a calendar reminder — passwords expire every 90 days.
Subsystem Not Active
System subsystem error
Alert IT immediately. Do NOT attempt to fix this yourself.
B311 Not Authorized
Register offline (Citi reward scan)
Rescan the Citi reward certificate. Override only after multiple attempts with system research completed.
