RF SCANNER — AS400 FIELD REFERENCE

Dept 61 · Meat Department · Warehouse Operations
All confirmed RF option codes · February 2026 Guide · Personal Use Only

The RF scanner is a full wireless AS400 session — same system, same data as the manager's office terminal, on your hip. Most floor staff use it as a barcode lookup. Supervisors use it as a running dashboard. Every option below is a skill gap between you and the next person on the floor. Data entry note: the numeric keypad Enter key blanks the rest of a field and tabs forward — faster than space-tab. Shift+F1–F12 = F13–F24.

Section 01
All RF Scanner Options
07
Print Depot Labels
Free-form text pallet labels
MEAT
Prints custom free-form text on pallet labels. You control what goes on the label. Used for staging, holds, and custom identification in the cooler.
How to run
  • 01Select option 07 on the scanner
  • 02Select printer: 1 = Receiving (RCV)  /  2 = RTV
  • 03Enter free-form text in the field
  • 04Enter number of tags to print → ENTER
⬛ Meat Dept Use

Label production holds, staged primal boxes, or overnight cut-prep. If a box of sub-primals is being held for a specific cut run, this label prevents it from being moved or confused with floor stock.

11
Item Inquiry
Most-used option — on-hand, sales, ordered, sell price
CRITICAL
DAILY
MEAT
Displays item detail by scanning a barcode or entering the item number. Shows department, on-hand in dollars and units, current week vs. previous week sales, ordered quantity, in-transfer, RTV, and sell price.
How to run
  • 01Select option 11
  • 02Scan barcode OR enter item number manually
  • 03Read the screen — all on-hand and sales data populates immediately
What the screen shows
  • DeptDepartment number
  • CWkCurrent week sales — dollars + units
  • PWkPrevious week — comparison baseline
  • OHOn-hand — current inventory in dollars + units
  • OrdOn order — what's coming in
  • TrnIn-transfer from another location
  • RTVReturn to vendor quantity
  • SELCurrent sell price
⬛ Meat Dept Use — YOUR PRODUCTION COMPASS

Run RF 11 on your top 5–10 movers before every shift. On-hand vs. current week units tells you if you're overstocked or running thin. If on-hand is high relative to this week's movement, cut less. If current week is burning through on-hand fast by mid-week, cut more and flag it. This is how a supervisor thinks — not just cutting by feel.

13
Item Alpha Inquiry
Search by item description — no barcode needed
MEAT
Look up items by description and/or department number. Two search fields, minimum 3 characters each required. Returns a list of matching items with department and item numbers.
How to run
  • 01Select option 13
  • 02Enter description keyword(s) — minimum 3 characters per field
  • 03Optionally enter department number to narrow results
  • 04Review the list and select your item
⬛ Meat Dept Use

New seasonal items, roadshow products, anything without a scannable barcode. Also useful for verifying a new item has been correctly set up in the system before first production.

16
UPC Inquiry
Barcode → item number lookup
MEAT
Scan or key a UPC barcode and get back the item description and item number. First step in diagnosing register scanning failures.
How to run
  • 01Select option 16
  • 02Scan the UPC barcode OR key it manually
  • KEYDrop the last digit when keying manually — that's the check digit
⬛ Meat Dept Use

When a package won't scan at the register or a scale label kicks back an error. Scan RF 16, verify the UPC is in the system and tied to the correct item. If the UPC is set up but still not scanning → EPC download from the AS400 terminal pushes the fix.

18
Item Sales Report
Print a sales report from the floor
MEAT
Creates and prints an item sales report for any set of items you specify. Scan or key each item you want on the report, then print it on F21.
How to run
  • 01Select option 18
  • 02Enter warehouse number
  • 03Enter date — optional, leave blank for current
  • 04Scan or key each item number you want included
  • F21Press F21 to print
⬛ Meat Dept Use

Pull a quick report on your high-velocity items at end of shift. Write today's on-hand counts on the printout. Keep a folder. That paper trail is what you show management when you talk production planning and when you push for advancement.

19
Item Group Code Maintenance
Build scan groups — batch your top items
UNDERUSED
MEAT
Create and maintain item groups — custom bundles of items you query together repeatedly. Build it once, use it forever. Eliminates scanning the same 20 items individually every single day.
How to run
  • 01Select option 19
  • 02Enter a group name OR press F4 to select an existing group
  • 03Press F4 to add items — scan or key UPC or item number
  • 04Press F10 to review the current item list
  • 05Key 4 next to any item to remove it → press ENTER
⬛ Meat Dept Use — MASSIVELY UNDERUSED

Create a group for your department's top 20 movers. Your daily on-hand check becomes a one-session scan instead of scanning 20 barcodes one by one. Most wrappers don't know this exists. Supervisors who use it look like they own the data.

20
Merchandise Audit Entry
Physical inventory count input
INVENTORY
CRITICAL
Scan or key item counts during physical inventory. Accepts multiple count entries per item and allows counts across multiple departments in one session.
How to run
  • 01Select option 20
  • 02Scan or key each item number
  • 03Enter your count for each item
  • 04Multiple entries for the same item are accepted — they stack
⬛ Meat Dept Use — CRITICAL ON COUNT DAY

Costco runs 13 physical inventory counts per year — one per 4-week fiscal period. Count day in meat is controlled chaos. The wrappers who know RF 20 get handed the scanner by the supervisor. Every count discrepancy that shows up later comes back to whoever was on the scanner. Own RF 20, own the count.

21
Clear Merchandise Audit Count List
Wipes ALL counts from RF 20 — irreversible
⚠ IRREVERSIBLE
Clears ALL counts entered in option RF 20. No partial clear. Everything or nothing.
How to run
  • 01Select option 21
  • 02Confirm to clear — counts are immediately gone
⚠ Critical Warning

Never hit RF 21 until RF 22 has printed and you have confirmed the report is correct. Firing this mid-count or before the report prints means the count is gone. You will have to recount. This mistake has burned people. The correct sequence is always: RF 20 → RF 22 → RF 21 (if clearing is needed at all).

22
Print Merchandise Audit Report
Print the count from RF 20 — same output as MADR
INVENTORY
Prints the count report from your RF 20 entries. Can print directly from the scanner or from the AS400 terminal. Produces the same report as AS400 option MADR.
How to run
  • 01Select option 22
  • 02Print from the RF scanner directly OR move to the AS400 terminal
MANDATORY SEQUENCE
RF 20
→
Enter all counts
RF 22
→
Print report — verify it's correct before moving on
RF 21
→
Clear list — only if required and only AFTER the report is confirmed
23
EDP Sign Requests
Print merchandise signs from the floor
MEAT
Print standard EDP price signs directly from the scanner without going back to the AS400 terminal. Specify sign size, scan items, send to printer.
How to run
  • 01Select option 23
  • 02Enter the letter for sign size — standard is S (small) unless directed otherwise
  • 03Enter item number(s) OR scan UPC(s)
  • F10Press F10 to review your item list before printing
  • 04Key 4 next to any item to remove it
  • F21Press F21 to print
  • 05Select the appropriate printer → ENTER
⬛ Meat Dept Use

Price changes on markdown items, sign corrections during promotion events. Run your own sign prints from the cooler without tying up someone at the office terminal. That kind of self-sufficiency gets noticed.

36
Print Pallet Tag
Pallet ID tags — receiving and RTV
MEAT
Prints pallet identification tags with item info, quantity, optional PO number, and receiver initials.
How to run
  • 01Select option 36
  • 02Select printer: 1 = RCV  /  2 = RTV
  • 03Enter item number OR scan UPC
  • 04Enter quantity
  • 05Enter PO number and initials — optional but strongly recommended
  • ENTPress ENTER to print
⬛ Meat Dept Use

Any primal shipment receiving or staging for RTV. PO number and your initials on every tag = an accountability chain. When a vendor dispute comes up — wrong count, wrong weight, wrong item — your tag is your proof. Supervisors who receive clean have far fewer disputes.

63
Fresh Foods Inventory (FFI)
Fresh dept physical inventory — 13× per year
INVENTORY
CRITICAL
MEAT
RF scanner access point for FFI (Fresh Foods Inventory — input counts). Specifically for fresh ancillary departments: Meat, Deli, Bakery, Seafood. The AS400 terminal equivalent is the FFI option. Prompts for an Aisle designation then walks through count entry.
How to run
  • 01Select option 63
  • 02Enter the Aisle identifier — cannot be blank, system will error
  • 03Enter counts by item
  • F11Press F11 to print from the RF scanner screen
⬛ Meat Dept Use — THIS ONE COUNTS TOWARD YOUR ADVANCEMENT

The fresh department physical count runs 13 times per year. Every variance between your RF 63 entries and the system's expected on-hand shows up directly in the department P&L. Supervisors are evaluated on shrink and inventory accuracy. If you own the count and the department's variance is tight, that is a direct, documentable win to put in front of management when you ask about advancement. Refer to the AS400 Options for Fresh Foods Inventory guide on the Inventory Audit page for the full fresh-dept count workflow.

72
Shrink
Remove damaged / expired items from inventory
WPM PASSWORD REQUIRED
HIGHEST CONSEQUENCE
Removes items from the warehouse's live inventory count when they are damaged, expired, or being disposed of. Requires a separate WPM-issued username and password — not your standard AS400 login. Credentials issued by WM or AWM only.
How to run
  • 01Select option 72
  • 02Enter WPM-issued username and password
  • 03Scan or key items being removed from inventory
  • 04Enter quantity in the Sell Unit field
Reversals
  • Same-day reversal: Key a negative amount in the Sell Unit field in the same session
  • Older reversals: Go through RTV using AS400 option NSE18
  • Research past entries: AS400 option TSH
⬛ Meat Dept Use — HIGHEST CONSEQUENCE OPTION ON THIS LIST

Every pound scanned out here hits your department's shrink number directly. In a high-volume meat department running thousands of pounds a week, shrink entries compound fast. That number shows up in the department P&L and managers see it. Running this option clean — with documentation on what you scanned and why — is exactly the kind of supervisor-level behavior that gets you promoted.

Hard Rules

Never scan out a pulled item without a physical tray or box in your hand.
Never reverse a shrink entry without understanding its effect on inventory.
Never let anyone else use your shrink credentials.

88
Item / UPC Cross Reference
All UPCs tied to one item — same as AS400 IXI
MEAT
Enter an item number and see every UPC currently associated with it. Same function as AS400 option IXI. Primarily used when an item isn't scanning at the register.
How to run
  • 01Select option 88
  • 02Enter the item number
  • 03View all UPCs tied to that item
  • KEYIf UPC is set up but not scanning → run EPC download from the AS400 terminal to push the fix
⬛ Meat Dept Use

Variable-weight meat items often have multiple PLUs and UPCs by pack size, grade, or label type. Tray of ribeyes won't scan at the register? RF 88 tells you immediately whether the UPC is in the system. Not set up = item master issue → WM or buyer. Set up but not scanning = EPC download.

98
Item / UPC Check
Item detail + sell price + multipack pricing
MEAT
View item description, item number, associated UPC, and sell price. If the item contains multiple sell units (multipacks), shows the individual per-unit pricing as well.
How to run
  • 01Select option 98
  • 02Scan or enter UPC OR item number
  • 03Read screen — description, item #, UPC, sell price, per-unit price for multipacks
⬛ Meat Dept Use

Quick price verification before a member escalates a complaint. Verify multipack pricing is set correctly — a 2-pack of Choice ribeyes should show both the total price and per-pack breakdown. Any discrepancy between what the sign says and what RF 98 shows goes to the office immediately.

Section 03
Career Priority — Meat Dept Track
01
RF 11 — DAILY
Run on your top items before every shift. Know the numbers before a manager asks. On-hand vs. current week = your production decision, made with data not instinct.
02
RF 63 + RF 20 — COUNT DAY
Own the inventory count. Be the person the supervisor hands the scanner to. A tight count is a direct, documentable win for your advancement conversation.
03
RF 72 — PUSH FOR ACCESS
Every dept has someone running Shrink. That person has elevated access and accountability. That's a supervisory function — push for it when trusted.
04
RF 19 — BUILD YOUR GROUPS
Build a top-20 meat item group once. Your daily check becomes one scan session. This is visible efficiency that supervisors and merchandise managers notice.
05
RF 36 + RF 07 — RECEIVE CLEAN
PO numbers and initials on every receiving tag. When vendor disputes happen, your documentation wins. Clean receiving = far fewer problems upstream.
06
RF 18, 22, 23 — LEARN, USE
Useful but secondary at this stage. Know them. Use them when needed. Not your primary differentiators in the first 6 months.
