    Daily Meat Routine

    Open

    1. Check top meat items in IAI or ITMW.
    2. Review yesterday movement and expected demand.
    3. Check on-hand, on-order, and delivery timing.
    4. Build cut list from movement, not guesswork.
    5. Verify signs before production ramps.

    Midday

    1. Re-check fast movers before they become out-of-stocks.
    2. Cut to fill need, not to keep the case overloaded.
    3. Watch slow movers and short shelf-life risk.
    4. Communicate one risk early: item, number, action.

    Close

    1. Record shrink by reason, not vague memory.
    2. Check whether receiving posted correctly in RHI.
    3. Save one win or issue in the Wins bank.
    4. Prep tomorrow’s cut note for the opener.

    Shrink Control

    • Watch repeated out-of-date shrink on the same item; that usually means overproduction or poor movement read.
    • Separate true trim/fat/bone loss from damages, leakers, out-of-date, donations, and sample/internal use.
    • Use item movement + days supply before cutting more.
    • Use receiving history to rule out mis-posted freight before blaming production.

    Meat Command Chain

    1. IAI / ITMW — item status, on-hand, on-order, ETA.
    2. IATOP / DSL — movement and department sales context.
    3. RHI — receiving history and posted quantity.
    4. SI / EDSR — sign verification and printing.
    5. WRKSPLF — save/report spool evidence.

    Receiving

    Verify freight, catch errors same day, protect inventory accuracy, and save proof.

    Daily Receiving Flow

    Before Truck

    1. Check DASH for inbound vendors, appointments, status, and doors.
    2. Know priority departments and likely problem freight.
    3. Prepare discrepancy notes before unload starts.

    During Truck

    1. Match PO / invoice / physical freight.
    2. Separate shortages, damages, wrong item, and overages immediately.
    3. For catch-weight freight, verify total weight tolerance using local receiving process.

    After Truck

    1. Use RHI to confirm posting after receiving.
    2. Use WRKSPLF for proof reports/spool references if available.
    3. Report variances the same day with item, PO, quantity, and action.
